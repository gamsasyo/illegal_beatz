# 한 컨트롤러 = 두 게임 (Dual-Control) — 작업 노트

> 신재영(f33dbag) 요청: "한 키보드로 WASD·SPACE 하면 비행기도 스케이트보드도 한 컨트롤 신호에 두 개가 동시에 움직여야 해. 정답 패스가 같아야겠지?"
> 렌이 밤새 작업한 1차 결과 + 다음 단계 분석.

## 1. 지금 되는 것 (dual.html)

`dual.html` 을 열면:
- 화면을 반으로 나눠 **왼쪽=skate.html, 오른쪽=flight.html** 를 iframe으로 동시 실행
- 최상위 페이지가 키보드를 잡고, **WASD · SPACE · E** 를 **두 게임에 똑같이 전달**(합성 KeyboardEvent)
- 두 게임 다 `?autostart=1` 로 클릭 없이 자동 시작
- iframe은 `pointer-events:none` → 클릭해도 포커스가 항상 최상위에 남아 한 입력이 양쪽으로

검증(Playwright): 최상위 window에 `KeyE` 디스패치 → HUD `AUTOPILOT`, 스케이트 `#ap`=`on`, 비행기도 오토파일럿. WASD/SPACE도 동일 경로로 양쪽 전달됨.

처음 한 번 **화면 클릭**(포커스 확보) 후 키 입력하면 됨.

## 2. 구조

```
dual.html (최상위, 키보드 포커스)
  ├─ keydown/keyup 캡처 → FWD=[W,A,S,D,Space,E] 만 골라
  │   두 iframe.contentWindow 에 new KeyboardEvent 디스패치 (repeat 플래그 보존)
  ├─ iframe a → skate.html?autostart=1&dual=1   (window 리스너로 수신)
  └─ iframe b → flight.html?autostart=1&dual=1
```
- 같은 origin(localhost) 이라 contentWindow 접근/디스패치 가능.
- 게임 둘 다 `addEventListener('keydown', …)` 를 window에 달아둬서 합성 이벤트가 그대로 먹음.
- skate.html 에 `autostart` 파라미터 분기 추가(클릭 오버레이 스킵). flight.html 은 원래 즉시 실행이라 변경 없음.

## 3. "정답 패스가 같아야" — 핵심 난제 분석

지금은 **각 게임이 자기 패스를 따로** 가진다:
- 스케이트 오토파일럿 = 트릭/그라인드 라인 (앞의 레일·킥커 조준, 올리, 공중 트릭)
- 비행 오토파일럿 = 링 패스 (wandering 곡선 위 16개 링)
→ E 누르면 둘 다 *각자* 자동으로 푼다. "한 신호 → 둘 다 성공"은 오토파일럿 레벨에선 이미 성립.

진짜 목표는 **수동 WASD 한 줄이 두 게임을 동시에 통과**시키는 것. 이게 어려운 이유:
| | 스케이트 | 비행기 |
|---|---|---|
| 주 입력축 | 타이밍(점프/그라인드) + 좌우 약간 | 좌우+상하 연속 추적(뱅크-투-턴) |
| A/D 의미 | 도로 위 좌우 스티어 | 롤(기울여 선회) |
| W/S | 가속/브레이크 | 피치(상/하) |
| Space | 올리(점프) | 부스트 |
| 실패 조건 | 레일 충돌/베일 | 링 놓침/장애물 |

→ 같은 키가 두 게임에서 **다른 물리**를 일으킴. 그래서 "아무 패스나" 양쪽에 통하진 않음.

### 풀이 방향(다음 세션 제안)
1. **공유 횡방향 패스 함수 `p(t)`** 도입
   - 비행 링과 스케이트 "수집/통과 게이트"를 **같은 `p(t)`** 좌우 좌표에 배치.
   - 두 게임 스티어 응답(감도·관성)을 맞춰서 `A/D` 로 `p(t)` 를 추적하면 둘 다 게이트 통과.
   - 스케이트는 좌우 추적이 약하니 → 도로에 **좌우 게이트(콘/링)** 를 추가해 비행과 같은 `p(t)` 로.
2. **결정론적 시드** — 두 게임 같은 seed로 레벨 생성(절차적 랜덤 제거) → 매판 같은 정답.
3. **타이밍 통합** — Space=비행 부스트 구간 ↔ 스케이트 점프 구간을 `p(t)` 의 "마커"로 동기화.
4. **고스트/정답 패스 표시** — 미리 만든 입력 스트림을 두 게임에 흘려보고(고스트), 플레이어가 따라하면 둘 다 클리어. (입력 녹화·재생 인프라 필요)

핵심: **둘을 "같은 추상 패스"에 종속**시키면(좌우 추적 + 타이밍 마커) 한 입력으로 동시 클리어 가능. 지금 harness(dual.html)가 그 위에 얹을 토대.
